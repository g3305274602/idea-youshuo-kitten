## 修正：我的頁「我的空間」誤開啟上一個他人空間

- 問題：在我的頁面點「我的空間」時，會沿用先前瀏覽他人空間留下的 `spaceTargetInfo`，導致仍顯示他人空間。
- 變更：
  - `src/features/app/hooks/useNavigationActions.ts`
    - `onMineHubNavigate("space")` 時，先 `setSpaceTargetInfo(null)`，再設定 `setSpaceOwnerHex(identityHex)`，確保回到自己的空間上下文。
    - 補上 `setSpaceTargetInfo` 參數型別。
  - `src/features/app/SpacetimeMailboxApp.tsx`
    - 傳入 `useNavigationActions` 的 `setSpaceTargetInfo` 參數。
- 結果：從我的頁點「我的空間」會穩定進入自己的空間，不再打開上一個他人空間。

## 修正：聊天進 TA 空間後返回無法回到原對話

- 問題：在聊天視窗中點「TA 的空間」後，回到聊天時無法回到原本那條對話。
- 變更：
  - `src/features/app/hooks/useMailboxInteractionHandlers.ts`
    - `openSpace` 新增 `fromChatThread` 判斷；若來源是 `chat`，不清除 `selectedChatThreadKey`，保留原對話上下文。
  - `src/features/app/hooks/useNavigationActions.ts`
    - `onMobileBack` 將 `activeTab === "space"` 的返回分支提前，先處理空間返回，避免先命中 `selectedChatThreadKey` 清空邏輯。
- 結果：聊天視窗進入 TA 空間後，返回會正確回到同一條對話。

## 修正：切換視窗回前景時誤觸發自動登出

- 問題：開啟其他視窗後回到頁面，因 `myProfile` 短暫未同步，啟動頁 1.2 秒自檢逾時會把使用者導向登入，造成「沒在其他裝置登入也被退出」。
- 變更：
  - `src/features/app/SpacetimeMailboxApp.tsx`
    - 在啟動自檢（有 token 但無 `myProfile`）加入 `document.visibilityState === "visible"` 條件，頁面不可見時不做逾時判定。
    - 將等待時間從 `1200ms` 調整為 `10000ms`，降低前景恢復時的誤判機率。
    - 移除該逾時分支中的 `clearLocalSessionState()`，避免短暫同步延遲就清 token 強制登出。
- 結果：切窗回來時不會再因短暫同步延遲而被自動登出；真正 session 失效仍由既有錯誤偵測流程處理。

## 修正：前景聚焦後偶發卡死無法重連

- 問題：回到前景時原本只重置連線等待 UI（`retryTick`），未必能讓底層 WebSocket 真正重建；若連線已僵住，會一直停在未連線狀態。
- 變更：
  - `src/App.tsx`
    - 新增 `connectedRef`、`identityRef` 追蹤即時連線狀態。
    - `visibilitychange` 回到 `visible` 時，先執行既有軟重連流程（`setRetryTick`）。
    - 再加一個 2.5 秒檢查：若仍未連上，觸發 `window.location.reload()` 做硬重連。
- 結果：切回前景時若軟重連失敗，會自動硬重整重建連線，避免長時間卡在「連不上」。
