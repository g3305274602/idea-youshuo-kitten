## 修正：刷新後無法留在管理頁（sessionStorage 被覆蓋／還原只跑一次）

- 問題：F5 後仍回到預設分頁。原因包含：(1) 還原 effect 開頭即標記「已還原」，在 `adminRole` 尚未載入、`auth.isAdmin` 仍為 false 時就結束，之後不再還原；(2) 持久化 effect 在 `myProfile` 就緒後立刻把預設 `activeTab`（如 `secret`）寫入 `sessionStorage`，覆蓋先前存的 `admin`／`admin_ops`；(3) 同一次 effect 週期內還原呼叫 `setActiveTab` 後，持久化仍讀到舊的 `activeTab`。
- 變更：
  - `src/features/admin/useAdminWorkbench.ts`：`useTable(tables.adminRole)` 解構出 `adminRoleRowsLoading`，一併回傳。
  - `src/features/app/SpacetimeMailboxApp.tsx`：以 `lastActiveTabRestoreCompletedRef` 控制「還原完成前不寫 sessionStorage」；管理分頁還原前等待 `myAccountId` 與 `!adminRoleRowsLoading`；以 `skipNextPersistAfterRestoreRef` 在還原套用分頁後略過下一次持久化。
- 結果：重新整理後可穩定回到管理分頁（權限符合時），且不會被首屏預設分頁覆寫儲存值。

## 追修：OTP 改為「註冊成功才消耗」，驗證成功不立即消耗

- 問題：先 `verify` 成功就刪除 challenge，導致使用者在 5 分鐘內再次輸入同碼時可能被要求重取驗證碼，體感不佳。
- 變更：`spacetimedb/src/index.ts`
  - `verify_email_otp` 成功後不再刪除 challenge，僅建立/刷新 `email_otp_session`，並重置 `attemptCount` 與 `lockedUntil`。
  - 新增 `consumeOtpChallengeForRegister()`，在 `register_account` / `register_account_with_email_otp` 註冊成功後才刪除 challenge。
  - 既有 `consumeOtpSessionForRegister()` 仍保留，確保 session 一次性消耗。
- 結果：驗證通過後到註冊提交前的等待期間，不會因提前消耗 challenge 而被迫重取；真正消耗點改到註冊成功時。
