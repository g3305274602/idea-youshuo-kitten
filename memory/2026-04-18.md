## SpacetimeDB 從倉庫根目錄建置

**原因**：`spacetime build` 預設只會找當前目錄下的 `spacetimedb/`，不會從 `spacetime.json` 讀取 `module-path`；模組實際在 `idea-jd-2-zx/spacetimedb`。

**處理**：

1. 在倉庫根目錄新增 `spacetime.json`（`server`、`database`、`module-path`、`dev.run`），`module-path` 為 `./idea-jd-2-zx/spacetimedb`；`dev.run` 使用 `npm --prefix idea-jd-2-zx run dev`，方便在根目錄執行與 Spacetime 搭配的前端指令。
2. 在根目錄新增 `package.json`，提供 `npm run spacetime:build` / `spacetime:dev` / `spacetime:generate` / `spacetime:publish`，並為 `build`/`generate`/`publish` 加上 `-p ./idea-jd-2-zx/spacetimedb`（`dev` 使用 `--project-path ./idea-jd-2-zx` 以使用子專案內既有設定）。

**使用方式**：在 `idea-youshuo` 根目錄請執行 `npm run spacetime:build`（或先 `cd idea-jd-2-zx` 再執行 `spacetime build`）。若堅持在根目錄直接打 `spacetime build`，需自行加上 `-p ./idea-jd-2-zx/spacetimedb`。

## 秘密膠囊／貼牆（產品規格定稿方向）

使用者定案：**僅此區分**——膠囊內＝私訊式往來；**貼牆後**才公開，公開內容與互動（含他人**評論**）依貼牆選項與牆規則，與膠囊聊天**不同通路**。實作時依此增量修改即可，不必改寫無關流程。

以下為已確認規則，**尚未**在 SpacetimeDB／前端全面實作；實作時需新欄位或新表與 reducer 權限校驗。

1. **膠囊主文可抽對象**  
   除**自己**外的人皆可抽到（與現有「排除自己」一致；其餘排除條件如已藏心底等維持產品決策）。

2. **膠囊內＝私訊式往來（與牆上「評論」分流）**  
   - 膠囊裡的回覆／聊天：**寄件與收件兩人**與**該位回覆者**之可見範圍依先前「獨立窗口」規則（私訊串）；**不等同**貼牆後的公開互動。  
   - **牆上**：僅在內容**貼到牆上**之後才進入**公開**語境；公開範圍依**貼牆當下**選項（僅主文／主文＋回覆／含全部膠囊聊天等由產品表列）。貼上後，**其他人可評論、按心情等**——此為**牆的公開評論**（沿用／對齊既有 `square_comment` 與互動模型），**與膠囊內私訊完全不同通路**。

3. **貼牆快照**  
   貼牆時可選是否納入膠囊私訊紀錄（納入多寡由貼牆選項決定）；一經貼牆，該快照與其後**公開評論**依牆規則；未貼牆前無此層公開評論。

**實作備註**：膠囊私訊表與 `square_post`／`square_comment` 分離；貼牆 reducer 依選項組快照字串或結構化欄位；訂閱範圍分「私訊參與者」與「廣場訂閱者」。
