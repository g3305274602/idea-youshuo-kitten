## 聊聊私訊氣泡誤判為「自己」（右側）

- **原因**：`ChatMainSection` 僅用 `authorAccountId === myAccountId` 判斷是否本人；若帳號欄位與預期不一致，對方訊息會被當成自己而靠右。
- **調整**：
  - `chatReadCursors.ts` 改為僅以 `authorAccountId` / `myAccountId`（雙側 `trim`）判斷本人訊息：`isChatMessageFromSelfByAccount`。
  - `computeThreadUnread` 同步改用帳號判斷，避免氣泡與未讀規則不一致。
  - `ChatMainSection` 氣泡左右改用 `isChatMessageFromSelfByAccount`，不再依賴 identity 欄位。

## 回覆他人膠囊時主文方向誤判

- **原因**：`isSourceCapsuleMine` 原本用 `threadGuestHex !== identity` 推導，會把「回覆他人膠囊」情境誤判為自己的主文，導致主文卡片顯示在右側。
- **調整**：改為以來源資料的 `accountId` 判斷：
  - 先找 `sourceMessageId` 對應的 `capsuleMessage.authorAccountId` 或 `squarePost.publisherAccountId`
  - 再與 `myAccountId`（`trim` 後）比較，僅相等時視為自己的主文。

  - 參與者判定：膠囊來源先比 `authorAccountId`（雙方非空才比較），若缺值則回退以 `authorEmail` 比對。

## SpacetimeDB 首次連線失敗自動清 token 重試

- **現象**：前端報 `WebSocket connection ... subscribe?compression=Gzip failed`，`main.tsx` 出現 `Error connecting to SpacetimeDB`。
- **排查結果**：CLI 可正常讀取 `idea-jd2zx` 日誌，主機/資料庫本身可達；高機率為瀏覽器保存的舊 `auth_token` 失效導致握手失敗。
- **調整檔案**：`src/main.tsx`
  - 新增 `RETRY_ONCE_KEY = ${HOST}/${DB_NAME}/connect_retry_once`（`sessionStorage`）。
  - `onConnectError` 中若偵測到有舊 token 且尚未重試：
    - 移除 `localStorage[TOKEN_KEY]`
    - 寫入 `sessionStorage[RETRY_ONCE_KEY] = "1"`
    - 自動 `window.location.reload()` 重試一次
  - `onConnect` 成功後清除 `RETRY_ONCE_KEY`，避免影響後續正常連線。
- **驗證**：
  - `npm exec tsc -- -b --pretty false` 通過
  - `ReadLints` 檢查 `main.tsx` 無新增警告/錯誤
